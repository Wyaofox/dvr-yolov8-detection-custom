# 基于YOLOv8的实时视频监控智能检测系统 — 项目说明稿

---

## 一、项目背景

随着安防需求的快速增长，传统视频监控系统依赖人工实时监控，存在效率低、易遗漏、成本高等问题。基于深度学习的目标检测技术为智能化视频监控提供了有效解决方案。

本项目基于开源项目 **dvr-yolov8-detection**（GPL-3.0 License）进行二次开发，实现了一套完整的实时视频监控智能检测系统。系统通过 USB 摄像头实时采集视频流，利用 YOLOv11 模型进行人体目标检测，结合自研的区域入侵检测算法，实现禁区监控、自动告警、视频录制等功能。

## 五、核心改动与创新点

### 5.1 区域入侵检测算法改进（重点创新）

**文件：** `zone_manager.py`（自研，全文约250行）

**原方案问题：**
- 使用 `sv.PolygonZone` 的 `triggering_anchors` 参数（中心点/多锚点触发）
- 只要人体任何一个锚点（头顶/脚底/左手/右手等）触碰禁区边界即触发告警
- 导致路过禁区边缘就误报，无法区分"经过"和"停留"

**我们的方案：**

1. **Bbox-Polygon 重叠度检测**（`_box_polygon_overlap()`, L22-55）
   - 计算检测框（bbox）与禁区多边形的交集面积占 bbox 面积的比例
   - 使用 OpenCV `fillPoly` 将多边形渲染为 mask，与 bbox mask 做 `logical_and` 计算交集
   - 重叠比例 ≥ 30% 才判定为"人在禁区内"

2. **停留计时器（Dwell Timer）**（`ZoneManager.update()`, L180-225）
   - 基于 tracker_id 追踪每个人的停留时间
   - 首次进入禁区时记录时间戳 `dwell_start[name][tid] = now`
   - 连续停留 ≥ `dwell_seconds`（默认3秒）才触发告警
   - 离开禁区后计时重置，同一人不会重复告警

3. **热加载（Hot-Reload）**（`ZoneManager.reload()`, L127-170）
   - 通过 NiceGUI 面板添加/删除区域后，立即生效无需重启程序

**优势对比：**

| 维度 | 原方案（锚点触发） | 我们的方案（重叠+停留） |
|------|-------------------|----------------------|
| 误报率 | 高（路过即触发） | 低（需停留3秒） |
| 检测精度 | 点级别 | 框级别（整体判断） |
| 可配置性 | 固定 | 重叠阈值/停留时间可调 |
| 抗干扰 | 弱 | 强（短暂经过不触发） |

## 六、关键代码讲解

### 代码段1：Bbox-Polygon 重叠度计算（核心算法）

**位置：** `zone_manager.py` L22-55

```python
def _box_polygon_overlap(box, polygon):
    """计算检测框与禁区多边形的重叠比例（交集 / 检测框面积）"""
    x1, y1, x2, y2 = box
    box_area = max((x2 - x1), 1) * max((y2 - y1), 1)
    if box_area == 0:
        return 0.0

    # 将禁区多边形渲染为二值 mask
    h, w = max(int(y2), 1) + 1, max(int(x2), 1) + 1
    mask = np.zeros((h, w), dtype=np.uint8)
    cv2.fillPoly(mask, [polygon.astype(np.int32)], 1)

    # 创建检测框的 mask
    bx1, by1 = max(int(x1), 0), max(int(y1), 0)
    bx2, by2 = min(int(x2), w), min(int(y2), h)
    box_mask = np.zeros_like(mask)
    box_mask[by1:by2, bx1:bx2] = 1

    # 计算交集像素数 / 检测框像素数
    intersection = np.logical_and(mask, box_mask).sum()
    return float(intersection) / float(box_area)
```

**讲解要点：**
1. 利用 OpenCV 的 `fillPoly` 将任意多边形渲染为像素级 mask
2. 通过 `logical_and` 计算两个 mask 的交集
3. 返回交集占检测框面积的比例（0.0~1.0），阈值可配置

### 代码段2：停留计时告警逻辑

**位置：** `zone_manager.py` L180-225

```python
def update(self, detections, fps=None):
    now = _time()
    zone_alerts = []

    for name, polygon in self.zones.items():
        active_tracks = set()

        for i in range(len(detections)):
            bbox = detections.xyxy[i]

            if self._bbox_in_zone(bbox, polygon):  # 重叠度 ≥ 阈值
                tid = int(tracker_ids[i])
                active_tracks.add(tid)

                # 首次进入，记录时间
                if tid not in self.dwell_start[name]:
                    self.dwell_start[name][tid] = now

                dwell = now - self.dwell_start[name][tid]

                # 停留超过阈值 且 未告警过 → 触发告警
                if dwell >= self.dwell_seconds and (name, tid) not in self._alerted:
                    self._alerted.add((name, tid))
                    zone_alerts.append((name, current_in_zone))

        # 离开禁区的人 → 清除计时，允许重新告警
        left_tracks = set(self.dwell_start[name].keys()) - active_tracks
        for tid in left_tracks:
            del self.dwell_start[name][tid]
            self._alerted.discard((name, tid))
```

**讲解要点：**
1. 使用 `tracker_id` 唯一标识每个人（基于 ByteTrack 跟踪器）
2. `dwell_start` 字典记录每人首次进入时间
3. `_alerted` 集合防止同一人重复告警
4. 离开后自动清除，可再次触发

## 八、总结

本项目在开源框架基础上，重点解决了实际安防场景中的三个核心问题：

1. **误报率高** → 通过 bbox-polygon 重叠度检测 + 停留计时器，将"经过"与"滞留"区分开来
2. **管理不便** → NiceGUI 统一面板，可视化区域编辑，热加载即时生效
3. **告警不实用** → 钉钉集成（国内场景）、自动录制（事后取证）

系统在 RTX 3050 Ti 上实现 21-26 FPS 的实时检测，满足实际监控需求。
